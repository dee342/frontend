Frontend SCSS & CSS

Frontend uses Compass and SASS to create and compile CSS stylesheets.

Framework SCSS Files

Global Styles (src/scss/_style.scss)

This file is an example of how to create individual CSS files within your application. Add/comment/uncomment the individual partials as needed.

Variables (src/scss/frontend/global/_variables.scss)

Frontend comes with a list of global variables used for theming. These can be helpful to keep site styles and branding consistent. Start by creating an application variables file running gulp variables.

Frontend Classes

Layout (src/scss/frontend/components/classes/_layout.scss)
  • .container: Used to set the maximum global width. should be used as a wrapper for the site's content.
  • .container-full: Used to set a site content wrapper to full-width.
  • .col-{$num-columns}-1: Used to define mobile & up full-width columns within a container wrapper. {$num-columns} should be replaced with the total number of grid columns for mobile devices & up.
  • .col-{$num-columns-small}-1-small: Used to define tablet & up full-width columns within a container wrapper. {$num-columns-small} should be replaced with the total number of grid columns for tablets & up.
  • .col-{$num-columns-medium}-1-medium: Used to define desktop & up full-width columns within a container wrapper. {$num-columns-medium} should be replaced with the total number of grid columns for desktops & up.
  • .col-{$num-columns-large}-1-large: Used to define large screens full-width columns within a container wrapper. {$num-columns-large} should be replaced with the total number of grid columns for large screens.
  • .clearfix: Clears a section of floated elements.
  • .flex: Defines a element to display as a flexbox.
  • .no-margin: Define an element's margin to 0.
  • .no-padding: Define an element's padding to 0.
